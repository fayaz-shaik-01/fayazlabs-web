## Why Linear Algebra Matters for AI

Every neural network is fundamentally a sequence of matrix multiplications and nonlinear transformations. Understanding linear algebra isn't optional — it's the language your models think in.

When you train a model, you're adjusting matrices of weights. When you compute attention, you're doing dot products. When you reduce dimensions with PCA, you're finding eigenvectors. Linear algebra is everywhere.

## Vectors: Direction and Magnitude

A vector is a quantity with both direction and magnitude. In AI, we use vectors to represent:

- **Data points**: Each feature is a dimension, so a data point with 768 features lives in 768-dimensional space
- **Embeddings**: Words, images, and concepts encoded as dense vectors
- **Gradients**: The direction of steepest loss increase during training

```python
import numpy as np

# A simple 3D vector
v = np.array([1, 2, 3])

# Vector magnitude (L2 norm)
magnitude = np.linalg.norm(v)  # √(1² + 2² + 3²) = √14 ≈ 3.74

# Unit vector (direction only)
unit_v = v / magnitude
```

## The Dot Product: Measuring Similarity

The dot product measures how aligned two vectors are. This is the foundation of attention mechanisms and similarity search.

```python
a = np.array([1, 0, 0])
b = np.array([0, 1, 0])
c = np.array([1, 0, 0])

# Orthogonal vectors: dot product = 0
print(np.dot(a, b))  # 0 — completely unrelated

# Identical direction: dot product = 1 (for unit vectors)
print(np.dot(a, c))  # 1 — perfectly aligned
```

**Cosine similarity** normalizes the dot product by magnitude, giving a value between -1 and 1. This is how embedding similarity is typically computed:

```python
def cosine_similarity(a, b):
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))
```

## Matrices as Transformations

A matrix isn't just a grid of numbers — it's a transformation. When you multiply a vector by a matrix, you're rotating, scaling, or projecting that vector into a new space.

```python
# Rotation matrix (90 degrees counterclockwise in 2D)
R = np.array([[0, -1],
              [1,  0]])

v = np.array([1, 0])
rotated = R @ v  # [0, 1] — rotated 90 degrees
```

In neural networks, the weight matrix of a linear layer is exactly this — a learned transformation that maps inputs to a new representation space.

## Matrix Multiplication: Composing Transformations

Matrix multiplication chains transformations together. If `A` maps from space X to Y, and `B` maps from Y to Z, then `BA` maps directly from X to Z.

```python
# Two transformations
A = np.array([[2, 0],
              [0, 3]])  # Scale x by 2, y by 3

B = np.array([[0, -1],
              [1,  0]])  # Rotate 90°

# Composed: first scale, then rotate
composed = B @ A

v = np.array([1, 1])
result = composed @ v  # Scale [1,1] → [2,3], then rotate → [-3, 2]
```

This is exactly what happens in a multi-layer neural network — each layer applies a transformation, and the composition of all layers is the model's overall function.

## Eigenvalues and Eigenvectors: The DNA of a Matrix

An eigenvector of a matrix `A` is a vector that, when transformed by `A`, only changes in magnitude — not direction. The scaling factor is the eigenvalue.

```python
A = np.array([[3, 1],
              [0, 2]])

eigenvalues, eigenvectors = np.linalg.eig(A)
# eigenvalues: [3, 2]
# The matrix stretches space by 3x along one axis and 2x along another
```

Eigendecomposition is the foundation of:
- **PCA**: Finding the directions of maximum variance
- **Spectral clustering**: Graph-based grouping
- **Understanding training dynamics**: How gradients flow through layers

## Building Intuition: What to Remember

1. **Vectors** represent points, directions, or features in high-dimensional space
2. **Dot products** measure similarity — this is the core of attention
3. **Matrices** are transformations — neural network layers are learned transformations
4. **Matrix multiplication** composes transformations — depth in networks means composed transformations
5. **Eigenvalues** reveal the intrinsic structure of a transformation

These aren't abstract concepts — they're the actual computations happening inside every model you'll build.
